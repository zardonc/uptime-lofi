<objective>
Add rate limiting to /setup and /unlock endpoints. These are currently unprotected administrative endpoints that allow setting/changing the dashboard password and disabling UI lock — critical vulnerability P0-4.

Purpose: Prevent brute-force attacks against the admin key (API_SECRET_KEY) used to access /setup and /unlock. Without rate limiting, an attacker can enumerate the admin key through unlimited attempts.
Output: Rate limit middleware applied to /setup and /unlock in api.ts routing layer.
</objective>
