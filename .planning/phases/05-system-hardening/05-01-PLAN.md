Purpose: Prevent rainbow table and brute-force attacks against stored password hashes. Single-iteration SHA-256 is cryptographically insufficient for password storage.
Output: PBKDF2-based crypto utilities with tests, updated auth and settings routes.
</objective>

<feature>
  <name>PBKDF2 Password Hashing</name>
  <files>backend/src/utils/crypto.ts, backend/src/utils/crypto.test.ts</files>
  <behavior>
    - hashPassword(password, salt) returns a hex string derived from PBKDF2-SHA256 with 10,000 iterations
    - verifyPassword(password, storedHash, salt) returns true when hash matches, false otherwise
    - Same password + same salt + same iterations always produces same hash (deterministic)
    - Different salt produces different hash for same password
    - Hashing takes approximately 5-15ms (10,000 iterations on Web Crypto API)
    - generateSalt(16) returns 32-char hex string (unchanged behavior)
  </behavior>
  <implementation>
    Use Web Crypto API's crypto.subtle.importKey + deriveBits for PBKDF2.
    Algorithm: PBKDF2 with SHA-256, 10,000 iterations, 32-byte output (256-bit key).
    Keep generateSalt unchanged — it already produces cryptographically secure random bytes.
    Remove calculateHash entirely — no callers should use it after migration.
  </implementation>
</feature>
